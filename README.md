# Bambi E-commerce System

Bambi is a multi-vendor e-commerce system built with TypeScript, Express, Prisma, and PostgreSQL. The repository contains both the backend API and the static storefront, seller, and admin UI served by the same server.

## Tech stack

- Node.js + TypeScript
- Express 5
- Prisma ORM
- PostgreSQL
- JWT authentication
- Static frontend assets in `public/`

## Main features

- Authentication and user profile management
- Product catalog, categories, and search
- Cart, checkout, and order management
- Shipment tracking, returns, and reviews
- Shop onboarding and seller tools
- Admin routes and admin UI
- Background jobs for auto-receive and seller payout processing

## Project structure

```text
.
|-- prisma/           Prisma schema and migrations
|-- public/           Storefront, account, seller, and admin UI assets
|-- src/
|   |-- jobs/         Scheduled background jobs
|   |-- libs/         Shared libraries
|   |-- middleware/   Express middleware
|   `-- modules/      Feature modules by domain
`-- dist/             Compiled output generated by `npm run build`
```

## Getting started

### 1. Install dependencies

```bash
npm install
```

### 2. Configure environment variables

Create a local `.env` file from `.env.example` and update the values for your machine.

Required:

- `DATABASE_URL`
- `JWT_SECRET`

Optional:

- `ACCESS_TOKEN_EXPIRES_IN`
- `REFRESH_TOKEN_TTL_DAYS`
- `PLATFORM_FEE_PERCENT`
- `SCHEDULER_ENABLED`
- `AUTO_RECEIVE_INTERVAL_MS`
- `PAYOUT_INTERVAL_MS`
- `GOOGLE_MAPS_API_KEY`
- `GOOGLE_MAPS_EMBED_KEY`

### 3. Generate Prisma client and run migrations

```bash
npm run prisma:generate
npx prisma migrate dev
```

### 4. Start the app

Development:

```bash
npm run dev
```

Production build:

```bash
npm run build
npm start
```

The API runs at `http://localhost:3000`.

## Default account

Default login for the current deployment:

- Email: `test@example.com`
- Password: `abc@123`

Important:

- This account will only be able to use the admin UI after its `users.role` is set to `admin`
- If the project was already deployed before that role was granted, run the SQL script below on production once
- After updating the role, sign out and sign in again so the new JWT contains the `admin` role

Promote the account on production:

```bash
psql "$DATABASE_URL" -f database/promote-test-admin.sql
```

Quick manual SQL:

```sql
UPDATE users
SET role = 'admin',
    updated_at = NOW()
WHERE email = 'test@example.com';

SELECT id, email, role, status
FROM users
WHERE email = 'test@example.com';
```

## Background jobs

`src/jobs/scheduler.ts` starts on boot, but only runs when `SCHEDULER_ENABLED=true`.

Current scheduled tasks:

- Auto-mark delivered items as received after the configured waiting period
- Release seller payouts for eligible received items
